# Getting Started with the Web Storage API in Vanilla JavaScript ## Building a Girls Act Register - Beginner Toolkit ## 1. Title and Objective **Chosen technology:** The Web Storage API (`localStorage`) with Vanilla JavaScript. **Why I chose it:** I wanted to learn how to build a small web application that works directly in the browser without needing a backend or database at the start. `localStorage` is a good beginner technology because it is simple to use, works in modern browsers, and makes it possible to save data between page refreshes. **End goal:** Build a simple participant register that: - collects participant details through an HTML form, - stores entries in the browser using `localStorage`, - displays saved entries in a table, - exports records to CSV and JSON. This project is available in my GitHub repository: `girls-act-register`. --- ## 2. Quick Summary of the Technology The **Web Storage API** is a browser feature that allows developers to store key-value data inside the browser. It includes: - `localStorage` - data stays saved even after the browser is closed, - `sessionStorage` - data is cleared when the tab or browser session ends. In this project, I used **`localStorage`** to keep participant records on the user's device. This made it possible to build a lightweight prototype without installing a database. ### Where it is used - saving dark/light theme preferences, - saving shopping cart data, - saving simple drafts or form entries, - prototyping small offline-first tools. ### One real-world example A website can store a user's language preference in `localStorage` so that the preferred language remains selected the next time the user opens the site. > Important note: `localStorage` is useful for learning and prototypes, but it is **not suitable for storing sensitive personal or health data in production**. --- ## 3. Common Issues and Fixes ### Issue 1: Data not saving after refresh **Cause:** I forgot that objects and arrays must be converted to strings before storing them. **Fix:** I used: ```javascript localStorage.setItem(STORAGE_KEY, JSON.stringify(entries)); ``` And when reading data back: ```javascript JSON.parse(localStorage.getItem(STORAGE_KEY) || '[]') ``` --- ### Issue 2: Export file downloads but is empty **Cause:** No entries had been saved yet. **Fix:** I added a check before exporting. If there are no entries, the app shows a message instead of downloading an empty file. --- ### Issue 3: Form clears, but table does not update correctly **Cause:** The app must re-render the table after saving or deleting records. **Fix:** I made sure `renderTable()` runs after saving, deleting, or clearing entries. --- ### Issue 4: GitHub Pages not showing the latest version **Cause:** GitHub Pages can take a short time to rebuild, or the wrong publishing branch may be selected. **Fix:** I checked: - the `main` branch is selected, - the root folder is selected, - the latest commit is pushed, - the page is refreshed after deployment finishes. --- ### Issue 5: Risk of storing sensitive data in a static site **Cause:** `localStorage` stores data in the browser without authentication or server-side protection. **Fix:** I clearly documented that this project is only for learning, prototyping, and non-sensitive demo data. A production version should use a secure backend and access control. --- ## 8. References ### Official documentation 1.
